## Immediate wins (1–3 days each)

### 1. Wilson confidence intervals on `reliability()` and hit rate
Add a `wilson_ci(k, n, z=1.96)` helper (4 lines of math) to `ceo_report.py`. Call it wherever `track_reliability()` returns a point estimate, and wherever `backtest.py` reports hit rates. Output format: `80% [67%, 89%] n=20`.

**Why:** Turns bare point estimates into statistically defensible numbers with no new data collection.
**Files:** `toolkit/ceo_report.py`, `toolkit/examples/backtest.py`

---

### 2. Fix `_parse_judge` — structured output enforcement
Replace the greedy `re.search(r"\{.*\}", content, re.DOTALL)` regex in `_parse_judge` with `json.loads()` + a schema check asserting `score ∈ [0.0, 1.0]` and `summary` exists. On failure, retry once with a stricter prompt suffix. Add a unit test covering nested JSON input.

**Why:** The current regex fails silently on nested JSON — wrong parse, no error raised.
**Files:** `toolkit/ceo_report.py`. New: `tests/test_parse_judge.py`

**Why:** Closes the `RIGOR NOTE` already in the docstring; "inter-rater kappa" is a named psychometric concept that reads well as portfolio evidence.
**Files:** `toolkit/ceo_report.py` (reference: `toolkit/examples/sync_dashboard.py`)
